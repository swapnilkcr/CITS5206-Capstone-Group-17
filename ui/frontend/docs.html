📂 Dataset Overview

All models are trained and evaluated on the SKAB (Skoltech Anomaly Benchmark) dataset — sensor readings from a water pump testbed. The dataset has 8 sensor channels sampled at fixed intervals.

Sensor columnsAccelerometer1RMS, Accelerometer2RMS, Current, Pressure, Temperature, Thermocouple, Voltage, Volume Flow RateRMS
Label columnanomaly — 0 = normal, 1 = anomalous
SeparatorSemicolon (;)
Total files35 labelled CSVs across data/valve1/, data/valve2/, data/other/
Anomaly-free baselinedata/anomaly-free/anomaly-free.csv — used to train unsupervised models
🎯 Supervised Models

Require labeled training data (anomaly column must be present). Trained per-upload using Leave-One-Out on all 35 SKAB files.

🎯 Supervised
XGBoost
Gradient-boosted decision trees · sklearn · 150 estimators
TRAINING STEPS
  1. Feature engineering Add rolling mean (window=5) and rolling std (window=5) for each of the 8 sensor channels → 24 total features. Apply StandardScaler.
  2. LOGO split When you upload a file, it is identified by MD5 hash. The model trains on the remaining 34 SKAB files and tests on your file only.
  3. Class imbalance handling Computes scale_pos_weight = n_normal / n_anomaly to prevent bias toward the majority class.
  4. Fit & predict Trains 150 trees (max_depth=6). Prediction threshold = 0.5 on predict_proba().
SPECS
Estimators150 trees
Max depth6
Features24 (8 raw + 8 roll_mean + 8 roll_std)
Threshold0.5 probability
EvaluationLOGO (single file) · Group K-Fold (ZIP)
✅ Best performer on in-distribution data. F1 ≈ 1.0 on familiar SKAB files.
⚠ Can overfit to training distribution. Needs labeled data at test time.
🎯 Supervised
Random Forest
Ensemble of decision trees · sklearn · 150 estimators · balanced class weight
TRAINING STEPS
  1. Feature engineeringSame 24 features + StandardScaler as XGBoost.
  2. LOGO splitSame Leave-One-Out procedure — trains on 34 files, tests on yours.
  3. Class imbalance handlingUses class_weight="balanced" — automatically adjusts sample weights per class.
  4. Fit & predict150 trees with bootstrap sampling. Aggregates votes across all trees. Threshold = 0.5.
SPECS
Estimators150 trees
Class weightbalanced
Features24
Threshold0.5 probability
✅ More stable than XGBoost across different file distributions.
⚠ Slightly lower recall — misses more faults but fewer false alarms.
🔍 Unsupervised Models

Trained only on normal (anomaly-free) data. No labels needed at test time — models learn what normal looks like and flag deviations.

🔍 Unsupervised
Isolation Forest
Random partitioning trees · trained once at startup on anomaly-free baseline
TRAINING STEPS
  1. Load anomaly-free baselineReads data/anomaly-free/anomaly-free.csv — zero anomaly rows, ever.
  2. Feature engineeringAdds rolling features → 24 features. StandardScaler fitted on baseline.
  3. Fit Isolation Forest300 trees, contamination=0.01. Anomalous points are isolated faster (fewer random splits needed).
  4. Compute thresholdScores all baseline rows: score = –decision_function(X). Threshold = 99th percentile of those baseline scores.
  5. Predict on new fileScore each row. Row is anomalous if score > threshold.
SPECS
Trees300
Contamination0.01 (1%)
Features24
Threshold99th pct of baseline scores
Training dataAnomaly-free baseline only
TrainedOnce at startup, cached
✅ No labels needed. Works on any file including unlabeled production data.
⚠ F1 ≈ 0.52 on SKAB. Cannot be fine-tuned with labeled data.
🔍 Unsupervised
LSTM Autoencoder
PyTorch · by Thant · 2-layer LSTM encoder-decoder · seq_len=30 · latent_dim=16
ARCHITECTURE
Input (30 timesteps × 24 features)
→ LSTM(24→64) → LSTM(64→16) [encoder]
→ LSTM(16→16) → LSTM(16→64) [decoder]
→ Linear(64→24) [reconstruct]
TRAINING STEPS
  1. Slice into windowsAnomaly-free baseline sliced into 30-row overlapping sequences.
  2. 80/20 train/val splitTrain up to 30 epochs, early stop (patience=5) on validation MSE.
  3. Compute thresholdReconstruction error (MSE) on all baseline windows. Threshold = 95th percentile.
  4. PredictSlide window across uploaded file. Window error > threshold → anomaly. Prediction assigned to the last row of each window.
SPECS
Window size30 timesteps
Latent dim16
OptimizerAdam, lr=1e-3
LossMSE (reconstruction)
Threshold95th pct of baseline errors
Features24 (with rolling)
TrainedOnce at startup, cached
✅ No labels needed. Good recall — catches most real faults.
⚠ F1 ≈ 0.52–0.55. High recall but moderate precision (some false alarms).
🔍 Unsupervised
Transformer Autoencoder
PyTorch · by Zeyang · self-attention encoder-decoder · seq_len=30 · d_model=32
ARCHITECTURE
Input (30 timesteps × 8 features)
→ Linear(8→32) + Positional Encoding
→ TransformerEncoder(d=32, heads=2, layers=1, ffn=64)
→ Linear(32→8) [reconstruct]
TRAINING STEPS
  1. Own scalerUses raw 8 features only (no rolling) with a dedicated StandardScaler.
  2. Slice & train30-row windows on anomaly-free baseline. Up to 50 epochs, early stop (patience=5).
  3. Threshold95th percentile of reconstruction MSE on baseline windows.
  4. PredictSliding window — self-attention processes all 30 timesteps simultaneously (non-sequential). High reconstruction error → anomaly.
SPECS
Window size30 timesteps
d_model32
Attention heads2
FFN dim64
Features8 raw only (own scaler)
Threshold95th pct of baseline errors
TrainedOnce at startup, cached
ℹ Captures long-range temporal dependencies within each 30-step window via self-attention, unlike LSTM's sequential processing.
⚠ Same F1 ≈ 0.52–0.55 as LSTM AE — performance is similar on SKAB.
⚡ Semi-supervised Models

Trained with labels, but no labels needed at test time. The best of both worlds — label signal guides training, but prediction is purely distance-based.

⚡ Semi-supervised
Adapted LSTM · ALSS-SVDD
PyTorch · by Julie · LSTM encoder + Deep SVDD · window=100 · hidden=64 · latent=32
ARCHITECTURE
Input (100 timesteps × 8 features)
→ LSTM(8→64)
→ Linear(64→32) [latent space]

No decoder — SVDD distance is the anomaly score.
TRAINING STEPS
  1. Load valve1 train splitJulie's valve1_train.csv (~12,713 rows, labeled normal + anomaly).
  2. Slide windows100-row windows, stride=10. Each window labeled 1 if any row in it is anomalous.
  3. Init SVDD centerMean embedding of all normal training windows in latent space.
  4. Train with 3 losses L1 (SVDD): Pull normal windows toward center.
    L2 (Margin): Push anomaly windows away from center (margin=2.0).
    L3 (Contrastive): Same-class windows similar, cross-class dissimilar (temp=0.2).
    Total = L1 + L2 + 0.5×L3.
  5. Threshold from val setAfter each epoch: score valve1_val.csv windows. Threshold = 95th percentile of normal window distances. Keep epoch with best val F1 (early stop patience=5).
  6. Predict (max-pooling)Slide window over test file. Each window's SVDD distance (distance² to center) is spread to all rows it covers via max-pooling. Row anomalous if its max score > threshold.
SPECS
Window size100 timesteps
Stride10
Hidden dim64
Latent dim32
OptimizerAdam, lr=1e-3
Max epochs20
SVDD margin2.0
Threshold95th pct of normal val window distances
Training datavalve1_train.csv + valve1_val.csv (labeled)
TrainedOnce at startup, cached
✅ Best overall — F1 = 0.894 on valve1 test, matching Julie's reported 0.873–0.895.
⚠ Threshold calibrated on valve1. Performance on valve2 (different sensor dynamics) is lower.

🌐 Real-world Deployment Guidance
⚠ Production data typically has no labels

In real-world water pump monitoring, incoming sensor streams do not come with anomaly = 0/1 labels. This has direct implications for which models are safe to deploy.

  • Do not use XGBoost or Random Forest in production — they require labeled data to be meaningful. Without labels you cannot evaluate or retrain them as conditions change.
  • Use Isolation Forest, LSTM AE, or Transformer AE for fully automated deployment — no labels ever needed at inference time.
  • Use Adapted LSTM when you have historical labeled fault data from the same pump type — re-train the SVDD encoder on your specific valve, then deploy label-free.
Recommended workflow for a new pump installation
1
Collect normal baseline data

Run the pump normally for a period. Capture this as your anomaly-free reference.

2
Train unsupervised models

Fine-tune Isolation Forest, LSTM AE, and Transformer AE on your normal baseline. Checkpoints are saved automatically.

3
Switch to inference mode

Upload new incoming sensor CSVs without labels. The model scores each row and flags anomalies in real time.

4
Label fault events when they occur

As faults are confirmed by engineers, label those windows. Accumulate enough labeled faults to fine-tune the Adapted LSTM for higher precision.

5
Retrain Adapted LSTM

Re-run training with the new labeled fault data. The SVDD center and threshold update to your specific pump's dynamics.

🔔 Unseen client data & retrain / fine-tune
Migration risk (honest disclaimer)

Checkpoints in this app were trained on the SKAB valve benchmark (fixed 8 sensor channels). If you upload a client CSV that is not in our SKAB library, the UI treats it as an unseen dataset. You can map your columns to the 8 SKAB names and run models for screening, but scores are not validated accuracy — sensor types, scaling, and pump dynamics may differ (domain shift).

For production use on your pumps, plan to retrain or fine-tune on your own normal baseline (and labelled faults when available).

Recommended retrain workflow
1
Collect client baseline CSVs

Export normal-operation sensor logs (no faults). Align timestamps and remove obvious bad rows.

2
Map or rename to 8 SKAB channels

In the app preview, map your columns to the 8 SKAB sensor names, or rename columns in your ETL pipeline.

3
Train unsupervised models on client normal data

Re-run notebooks under pre_trained_model/ipynb_scripts/ for Isolation Forest, LSTM AE, and Transformer AE.

4
Export checkpoints

Place artifacts under pre_trained_model/artifacts/ and register in registry.yaml.

5
Fine-tune Adapted LSTM when labels exist

After confirmed faults, add anomaly labels and re-train ALSS-SVDD on your valve data.

UI: upload unseen CSV → map 8 columns → migration-risk popup → run unsupervised model for screening.
